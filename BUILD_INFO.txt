File Descriptions
-----------------
1. tvm.dll (109 MB)
   - Full TVM library with LLVM codegen support
   - Required for model compilation and conversion
   - Contains LLVM-based code generation for x64 targets

2. tvm_runtime.dll (2.5 MB)
   - Lightweight TVM runtime (no LLVM)
   - Used for model inference only
   - Smaller footprint for production deployments

3. tvm_ffi.dll (1.4 MB)
   - TVM Foreign Function Interface
   - Python/C++ interop layer
   - Required for Python bindings

4. tvm_ffi_testing.dll (582 KB)
   - Testing utilities for TVM FFI
   - Optional for most use cases

What Makes This Build Special
------------------------------
This build includes LLVM support, which is REQUIRED for:
- MLC-LLM model weight conversion (convert_weight command)
- MLC-LLM config generation (gen_config command)
- CPU-based model compilation
- LLVM IR code generation

The official MLC-LLM pre-built Windows wheels DO NOT include LLVM support,
making model conversion impossible on Windows without building from source.

Build Challenges Solved
------------------------
1. MinGW vs MSVC compatibility issues
2. LLVM development SDK installation (headers + libs)
3. libxml2 MSVC format requirements (via vcpkg)
4. CMake 4.x compatibility with msgpack and sentencepiece
5. Missing posix_memalign function in MinGW builds

Tested Use Cases
----------------
✅ MLC-LLM model weight conversion (Qwen 2.5 3B tested)
✅ MLC-LLM config generation
✅ TVM Python package integration
✅ Model quantization (q4f16_1 tested successfully)
